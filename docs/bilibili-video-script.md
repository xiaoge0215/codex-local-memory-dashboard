# 哔哩教学视频分镜脚本

## 视频定位

- 标题建议：我给 Codex 装了一个本地记忆脑，还能自动生成每日工作看板
- 目标观众：Codex / Claude Code / Cursor / MCP 用户、AI 工具玩家、个人自动化用户
- 视频时长：8–12 分钟
- 风格：高级感桌面演示 + CEO 口播 + 屏幕录制 + 字幕强调

## 核心卖点

1. 数据留在本地，不依赖云端。
2. `AGENTS.md` 固化项目规则和个人习惯。
3. `agent-memory-mcp` 保存长期记忆。
4. `LEARNINGS.md` 让 bug 修复变成可复用经验。
5. 每日产出看板自动整理 Codex 产出。
6. 支持按项目筛选、核心内容过滤、跨天会话归档。
7. 零依赖 Node 服务，低配电脑也能按需启动。

## 分镜

### 00:00–00:20 开场钩子
- 画面：桌面快速切换 Codex、看板、AGENTS.md、LEARNINGS.md。
- 旁白：你有没有发现，AI 编程助手很聪明，但它经常忘事？同一个坑，下次还会踩。今天我们给 Codex 搭一个本地记忆脑。
- 字幕：本地记忆 + 自我进化 + 每日产出看板

### 00:20–01:10 痛点展示
- 画面：三张卡片：上下文丢失、bug 重复、产出难复盘。
- 旁白：普通聊天记录不等于工程记忆。真正有用的是规则、决策、踩坑、修复方案和每天的产出沉淀。
- 互动：弹幕提问：你有没有遇到 AI 重复犯同一个错误？

### 01:10–02:10 方案架构
- 画面：四层架构图：AGENTS.md → agent-memory-mcp → LEARNINGS.md → Daily Dashboard。
- 旁白：规则文件约束行为，MCP 记忆库负责长期检索，LEARNINGS 负责人类可读台账，看板负责复盘产出。
- 字幕：不是替代 Codex，而是给 Codex 加工作记忆。

### 02:10–03:30 AGENTS.md + 记忆规则
- 画面：打开 AGENTS.example.md，突出“开工前检索”“收尾时沉淀”。
- 旁白：每个任务开始前先看最近踩坑，再查记忆库。任务结束后，如果修了 bug、被纠正、形成模式，就写入 LEARNINGS 和 memory。
- 展示：projectId、memory_write、bug/pattern/decision/feature。

### 03:30–04:40 自我进化台账 LEARNINGS
- 画面：展示 LEARNINGS 示例条目。
- 旁白：LEARNINGS 是给人看的，memory 是给 Agent 检索的。两者结合，把一次修复变成长期能力。
- 展示：跨天记录不显示如何沉淀成 bug 经验。

### 04:40–06:30 每日产出看板演示
- 画面：打开 `http://localhost:3455`。
- 展示：汇总卡、按天折叠、会话详情、按项目筛选。
- 旁白：看板直接读取本地 `~/.codex/sessions`，不上传数据。它按天整理提问、回复、命令、文件修改。
- 互动：让观众猜“全部记录”和“核心内容”差多少。

### 06:30–07:40 核心内容过滤
- 画面：切换“核心内容 / 全部记录”。
- 展示：回复、命令、文件修改数量明显下降。
- 旁白：真正的日报不应该塞满系统注入和探测命令。核心模式会过滤短进度播报、临时文件、端口检查、进程检查，只保留更像产出的内容。
- 字幕：不是删数据，是默认降噪；全部记录随时可查。

### 07:40–08:50 桌面双击启动 + 低配优化
- 画面：双击桌面图标打开看板。
- 展示：`.bat` → `start-dashboard.ps1` → `server.js`。
- 旁白：为了低配电脑不卡，这个看板没有数据库，没有前端框架，没有第三方依赖。用的时候双击，不用时关掉。
- 字幕：零依赖 Node 服务，适合本地按需启动。

### 08:50–10:00 GitHub 项目发布
- 画面：展示 GitHub README、目录结构、示例 AGENTS、示例 LEARNINGS。
- 旁白：发布到 GitHub 的一定是脱敏版。不要上传自己的 sessions、记忆库数据、本机路径和私有项目内容。
- 字幕：开源代码可以分享，私人记忆不要上传。

### 10:00–10:40 结尾 CTA
- 画面：看板最终效果 + GitHub 仓库页面。
- 旁白：如果你也想让 AI 编程助手越用越懂你，可以试试这套本地记忆工作流。需要我继续做一版跨平台安装脚本，可以在评论区告诉我。
- 互动：评论区投票：你更想要“日报导出”还是“自动生成周报”？

## 高级展示建议

- 用暗色主题屏幕录制，窗口边缘加柔和阴影。
- 关键数字用放大动画：例如“命令 4709 → 2005”“文件修改 1408 → 147”。
- 用 Mermaid 或手绘风架构图解释四层方案。
- 展示真实功能时只用脱敏项目名，避免暴露本机路径。
- CEO 口播可以采用“产品发布会”语气：问题、方案、证明、开源。
