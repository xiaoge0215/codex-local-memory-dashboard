# 配音字幕稿

你有没有发现，AI 编程助手很聪明，但它经常忘事。
同一个 bug，今天修过了，下次它可能还会踩。
所以我做了一套本地 Codex 记忆工作流。

它由四部分组成。
第一层是 AGENTS.md，用来固定项目规则、技术栈和我的编码习惯。
第二层是 agent-memory-mcp，用来保存长期记忆、bug 修复和技术决策。
第三层是 LEARNINGS.md，给人看的踩坑台账。
第四层是每日产出看板，自动读取本地 Codex sessions，整理每天做了什么。

这套系统最大的特点是：数据都在本地。
它不会把你的会话上传到云端，也不需要数据库。
看板本身是一个零依赖 Node 服务，低配电脑也可以按需启动。

看板支持按天查看、按项目筛选、点击会话看详情。
而且我专门加了核心内容过滤。
因为 Codex 的原始记录里有很多系统注入、短进度播报、端口检查、进程检查、临时文件操作。
如果全部统计，数据会很虚。
核心模式会默认降噪，只保留更像产出的内容。
需要审计的时候，再切回全部记录。

我还修了一个很真实的问题。
如果昨天开的会话今天继续聊，旧版看板会把今天的内容算到昨天。
现在改成按最近活动时间归档，所以跨天会话也不会漏进今天日报。

最后，发布到 GitHub 的版本是脱敏版。
代码可以开源，但自己的 sessions、记忆库、本机路径、私有项目内容不要上传。

如果你也想让 Codex 越用越懂你，可以试试这套本地记忆脑。
下一期我可以继续做：自动导出日报，或者自动生成周报。
你更想看哪个，评论区告诉我。
